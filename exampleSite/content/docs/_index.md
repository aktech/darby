Welcome to **Darby Docs**, the demo site for a beautiful, minimal Hugo documentation theme. It gives you everything you need to publish polished technical documentation: a clean reading layout, instant dark and light modes, client-side search, syntax-highlighted code blocks, and a collection of shortcodes for callouts, tabs, cards, and step-by-step guides.

This site is itself built with the theme, so every page you read here is also a live example of what the theme produces.

## What you get

The theme is designed to stay out of your way. You write Markdown, drop in a few shortcodes when you need them, and the theme handles typography, navigation, and responsiveness for you.

## Key features

- **Dark and light modes** that respect the visitor's system preference and remember manual choices.
- **Configurable fonts** for body, headings, and monospace text, with Fira Code as the default code font.
- **A single accent color** that flows through links, buttons, the active navigation item, and focus rings.
- **Client-side search** with no external service required.
- **Callouts and GitHub-style alerts** for notes, tips, warnings, and cautions.
- **Tabs, code groups, cards, and steps** so you can present information the way it reads best.
- **A version switcher and top-level tabs** for organizing large documentation sets.

## Why this theme

Most documentation themes force a tradeoff: either they look good but are hard to customize, or they are flexible but ship with cluttered defaults. This theme aims for both. The defaults are deliberately quiet, so your content is the loudest thing on the page, and the handful of parameters you can change (accent, fonts, search) cover the vast majority of branding needs without touching a single template.

Because it is a standard Hugo theme, you keep all of Hugo's speed and portability. There is no Node build step, no proprietary content format, and no lock-in: your content is plain Markdown that you can move anywhere.
