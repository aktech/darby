A theme can only do so much; the writing carries the rest. A few habits make
documentation noticeably easier to follow.

Lead with the task, not the tour. Readers arrive with a goal, so open with what
they are trying to do and get them to the first working result quickly.

Show the result next to the instruction. Darby's tabs and code groups let you
put the source and its rendered output side by side, which removes a lot of
guesswork for the reader.

Keep one idea per callout. A page where everything is highlighted has nothing
highlighted. Reserve warnings for the things that will actually break, and let
the prose carry the rest.
